## Context

当前应用支持中文、英文、法语三种语言，但法语翻译不完整。`src/locales/fr/setting.json` 缺少 3 个键值，导致法语用户可能看到未翻译的文本。现有的国际化系统使用 i18next，翻译文件存储在 `src/locales/<lang>/` 目录下，每个模块一个 JSON 文件。

当前没有自动化机制来验证翻译完整性，依赖人工检查，容易遗漏。

## Goals / Non-Goals

**Goals:**
- 补充法语翻译中缺失的 3 个键值（`autoNaming.title`、`autoNaming.description`、`languageSwitchFailed`）
- 建立自动化翻译完整性检查机制，防止未来出现遗漏
- 提供清晰的翻译差异报告，帮助开发者快速定位问题

**Non-Goals:**
- 不修改现有的 i18next 架构
- 不改变翻译文件的组织结构
- 不实现自动翻译功能（翻译内容仍需人工提供）

## Decisions

### 1. 翻译检查工具实现方式

**决策**：使用 Node.js 脚本作为翻译检查工具

**理由**：
- ✅ 无需引入新的运行时依赖（项目已有 Node.js 环境）
- ✅ 可以轻松集成到现有 npm scripts 中
- ✅ 可以使用现有的 JSON 处理库，无需额外依赖
- ✅ 跨平台兼容性好

**替代方案**：
- Shell 脚本：JSON 处理复杂，可读性差
- Python 脚本：需要额外依赖，增加项目复杂度

### 2. 基准语言选择

**决策**：使用英文作为基准语言（reference language）

**理由**：
- ✅ 英文是最初实现的语言，文件最完整
- ✅ 项目约定使用英文作为"第一公民"（见 AGENTS.md）
- ✅ 符合国际化最佳实践

### 3. 检查工具的执行时机

**决策**：同时支持手动执行和 Git Hook 自动执行

**理由**：
- ✅ 手动执行：开发者可以随时检查，开发体验好
- ✅ Git Hook：在提交前自动检查，防止遗漏进入代码库
- ✅ 可配置性：开发者可以选择性启用 Git Hook

### 4. 错误处理策略

**决策**：发现缺失翻译时，工具以非零退出码退出

**理由**：
- ✅ 可以在 CI/CD pipeline 中作为失败条件
- ✅ 在 Git Hook 中可以阻止提交
- ✅ 提供清晰的错误信息，指出具体缺失的键值

### 5. 翻译文件结构

**决策**：维持现有的模块化翻译文件结构

**理由**：
- ✅ 现有结构运行良好
- ✅ 按功能模块划分（common, chat, navigation 等）易于维护
- ✅ 无需重构现有代码

## Risks / Trade-offs

### 风险 1：误报（False Positives）
**描述**：某些语言可能有意不翻译某些键值（如 `itemsUnit` 在英文和法语中为空字符串）

**缓解措施**：
- 在检查工具中添加"允许的空值列表"配置
- 允许开发者通过配置文件排除特定键值
- 在文档中明确说明如何处理合法的空值

### 风险 2：性能影响
**描述**：翻译检查可能增加 CI/CD pipeline 的执行时间

**缓解措施**：
- 检查工具非常轻量，预计执行时间 < 100ms
- 只在修改翻译文件时运行检查（通过 Git staging area 检测）
- 提供跳过选项（`--skip-i18n-check`）用于紧急情况

### 风险 3：法语翻译质量
**描述**：新增的法语翻译可能不够地道

**缓解措施**：
- 由熟悉法语的开发者或翻译工具提供翻译
- 在实施过程中邀请法语使用者进行审查
- 记录翻译来源，便于后续改进

## Migration Plan

### 阶段 1：补充缺失翻译
1. 在 `src/locales/fr/setting.json` 中添加 3 个缺失的键值
2. 验证法语翻译在应用中正常显示

### 阶段 2：创建检查工具
1. 创建 `scripts/check-i18n.js` 脚本
2. 实现键值比较逻辑
3. 生成清晰的差异报告

### 阶段 3：集成到开发流程
1. 在 `package.json` 中添加 `lint:i18n` 和 `validate` scripts
2. 可选：配置 Husky Git Hook
3. 更新 AGENTS.md 和 README.md（如有必要）

### 阶段 4：验证和文档
1. 运行检查工具验证所有语言文件
2. 编写工具使用文档
3. 在团队中推广使用

### 回滚策略
如果新的翻译有问题：
- 直接回滚 `src/locales/fr/setting.json` 的更改
- 移除或禁用 Git Hook（如已配置）
- 保留检查工具代码，标记为实验性功能

## Open Questions

**Q1**: 是否需要支持嵌套键值的检查？
- 当前假设：翻译文件只有一层嵌套（如 `autoNaming.title`）
- 待确认：是否有更深层次的嵌套需要处理

**Q2**: 检查工具是否应该支持自动修复（自动添加空值）？
- 当前决策：只检查不修复，避免误操作
- 可选：未来添加 `--fix` 选项自动填充空值

**Q3**: 是否需要忽略某些文件或键值？
- 当前决策：暂不考虑，所有键值必须完整
- 可选：未来添加 `.i18nignore` 配置文件
