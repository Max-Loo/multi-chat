## Context

项目经过多轮迭代后，积累了测试基础设施残留（MSW 迁移后遗留）、过度导出的 UI 组件和 fixture 工厂函数。knip 工具检出 89 个未使用导出、5 个未使用文件和 1 个未使用依赖。当前无运行时影响，但增加维护复杂度。

## Goals / Non-Goals

**Goals:**
- 清理 knip 检出的所有未使用代码（文件、导出、依赖、枚举成员）
- 精简 knip.json 配置，减少误报噪音
- 确保清理后项目构建和测试通过

**Non-Goals:**
- 不重构任何现有代码结构或架构
- 不清理 TypeScript 类型/接口导出（它们是 API 设计的一部分，保留供未来使用）
- 不修改任何运行时行为

## Decisions

### 决策 1：分三个优先级批次执行清理

**选择**：按影响范围分批次（立即清理 → 推荐清理 → 可选审查）
**理由**：未使用的测试 fixture 和依赖删除风险最低，优先处理；UI 组件导出需确认无动态引用；类型导出保留。

| 批次 | 内容 | 风险 |
|------|------|------|
| P0 | 删除未使用文件、移除 msw 依赖 | 极低 |
| P1 | 清理未使用的 UI 组件导出、枚举成员、重复导出 | 低 |
| P2 | 精简 knip.json 配置 | 极低 |

**替代方案**：一次性删除所有 — 被否决，因为 89 个导出中部分可能是动态引用，需逐一确认。

### 决策 2：未使用的测试 fixture 全部删除

**选择**：删除 `__test__/fixtures/` 和 `__test__/helpers/` 中未被引用的 25 个导出
**理由**：这些 fixture 是之前 MSW 测试策略的产物，当前无任何测试文件引用它们。如果未来需要，可以从 git 历史恢复。

### 决策 3：TypeScript 类型/接口导出保留不清理

**选择**：保留所有 40+ 个未使用的类型/接口导出
**理由**：这些是模块公共 API 的一部分，删除会降低模块的可扩展性。knip 的类型导出误报率较高，不值得逐一审查。

## Risks / Trade-offs

- **[动态导入遗漏]** → 某些导出可能通过动态路径引用，knip 无法检出 → 清理后运行完整测试套件验证
- **[误删 fixture]** → 删除的 fixture 可能在未来的测试开发中需要 → 可从 git 历史恢复
- **[枚举成员删除]** → 删除 `PARSE_ERROR` 和 `ABORTED` 后，如果有序列化数据引用这些值 → 风险极低，这些是内部错误类型
