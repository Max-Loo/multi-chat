# Specs: 国际化架构重构

## 概述

本次重构不涉及新的能力定义或现有需求变更，因此**不需要创建新的 spec 文件**。

## 原因

根据 `proposal.md` 的 **Capabilities** 部分：

### New Capabilities
无（本次重构不引入新的用户可见能力）

### Modified Capabilities
无（本次重构不改变 `i18n-lazy-loading` 规范中的任何需求，仅优化实现架构）

## 重构范围

本次重构聚焦于**实现架构优化**，不改变用户可见的行为或 API 契约：

- 统一语言配置源（内部实现细节）
- Toast 消息队列机制（内部实现细节）
- 自动语言持久化（内部实现细节）

所有这些变更都在现有的 `i18n-lazy-loading` 规范框架内，不改变任何需求场景。

## 相关规范

本次重构继续满足以下现有规范的要求：

- **openspec/specs/i18n-lazy-loading/spec.md** - 定义了语言懒加载的所有需求场景，包括：
  - 英文资源同步打包为"第一公民"
  - 系统启动时按需加载系统语言
  - 语言切换时动态加载目标语言
  - 语言资源加载失败时自动重试
  - 语言资源加载缓存机制
  - 语言加载状态的用户反馈

重构后的实现将继续满足这些需求，仅在内部架构上更加清晰和可维护。
