# Canvas 动态 Logo - 技术设计文档

## Context

### 当前状态

初始化页面 (`InitializationScreen`) 使用 shadcn/ui 的 Skeleton 组件显示简单的脉冲动画，缺乏品牌特色。应用现有 logo (`public/logo.svg`) 是一个复杂的机器人 SVG，不适合小尺寸显示。

### 设计需求

基于用户讨论确定的设计方向：

| 属性 | 选择 |
|------|------|
| 风格 | 极简现代 |
| 机器人形状 | 圆角方形 |
| 眼睛样式 | LED 像素风格 |
| 场景 | 桌前思考打字 |
| 打字内容 | 键盘按键动画 |
| 配色 | 黑白线条 + 蓝色点缀 |
| 动画复杂度 | 丰富 (10+ 元素) |

### 约束条件

- 无新增外部依赖
- 必须支持 `prefers-reduced-motion`
- 兼顾初始化页面动画和 App 图标两种用途

## Goals / Non-Goals

**Goals:**

1. 创建流畅的 Canvas 动画，展示机器人思考打字场景
2. 设计统一的视觉语言，确保动画和静态图标风格一致
3. 实现高性能动画，保持 60fps
4. 输出静态 SVG 图标，适合小尺寸显示

**Non-Goals:**

- 不实现交互式动画（点击、悬停等）
- 不支持用户自定义动画参数
- 不实现音频效果
- 不创建多个主题变体（仅固定配色）

## Decisions

### D1: 组件架构

**决策**: 采用三层架构

```
┌─────────────────────────────────────────────────────────────┐
│                    组件架构                                  │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  InitializationScreen/index.tsx                             │
│         │                                                   │
│         ▼                                                   │
│  AnimatedLogo.tsx (React 组件)                              │
│    - 管理 Canvas 生命周期                                   │
│    - 响应 prefers-reduced-motion                            │
│    - 处理尺寸变化                                           │
│         │                                                   │
│         ▼                                                   │
│  canvas-logo.ts (纯 Canvas 逻辑)│
│    - 动画状态管理                                           │
│    - 绘制函数                                               │
│    - requestAnimationFrame 循环                             │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

**理由**: 分离关注点，React 组件处理生命周期，纯 TS 文件处理动画逻辑，便于测试和维护。

### D2: 动画元素设计

**决策**: 实现 10 个动画元素

| # | 元素 | 动画效果 | 实现方式 |
|---|------|---------|---------|
| 1 | 思考气泡 | 三点依次跳动 | sin 函数控制 Y 偏移 |
| 2 | LED 眼睛 | 闪烁/亮度变化 | 透明度变化 + 像素矩阵 |
| 3 | 天线 | 左右轻微晃动 | sin 函数控制旋转角度 |
| 4 | 头部 | 轻微左右倾斜 | sin 函数控制旋转 |
| 5 | 身体 | 呼吸效果 | sin 函数控制 Y 偏移 |
| 6 | 打字手臂 | 左右交替按键 | sin 函数控制 Y 偏移 |
| 7 | 键盘 | 底座绘制 | 静态矩形 |
| 8 | 按键 | 依次激活高亮 | 时间计数器控制激活索引 |
| 9 | 手指 | 跟随按键移动 | 手臂动画同步 |
| 10 | 整体 | 轻微漂浮感 | sin 函数控制整体 Y 偏移 |

**理由**: 10 个元素足够丰富，但不过于复杂。每个元素使用独立的相位，创造自然的异步效果。

### D4: 配色方案

**决策**: 黑白为主，蓝色点缀

| 元素 | 颜色 | 说明 |
|------|------|------|
| 背景 | 透明 | 适应任何背景 |
| 机器人轮廓 | `#333333` | 深灰，2px 线条 |
| 机器人填充 | `#FFFFFF` | 白色 |
| LED 眼睛 | `#4269C4` | 品牌蓝 |
| 聊天气泡 | `#4269C4` | 品牌蓝 |
| 键盘 | `#E5E5E5` | 浅灰 |
| 按键 | `#FFFFFF` | 白色 |
| 桌面线条 | `#666666` | 中灰 |

**理由**: 极简配色方案，蓝色点缀强调品牌识别度，确保在深色和浅色背景下都清晰可见。

**理由**: 小尺寸下复杂细节会模糊，保留识别度最高的核心元素。

### D6: 性能优化

**决策**: 多层优化策略

1. **动画循环**: 使用 `requestAnimationFrame`
2. **减少重绘**: 只在需要时更新状态
3. **清理机制**: 组件卸载时取消动画帧
4. **无障碍**: 检测 `prefers-reduced-motion`，提供静态版本

```typescript
// 检测 prefers-reduced-motion
const prefersReducedMotion = window.matchMedia(
  '(prefers-reduced-motion: reduce)'
).matches;

// 如果用户偏好减少动画，只绘制静态帧
if (prefersReducedMotion) {
  drawStaticFrame(ctx);
  return;
}
```

**理由**: 确保动画在各种设备上流畅运行，同时尊重用户的无障碍偏好。

## Risks / Trade-offs

### R1: Canvas 在某些环境下可能不可用

**风险**: Tauri WebView 或旧版浏览器可能不完全支持 Canvas API。

**缓解**: 添加 Canvas 支持检测，降级到静态 SVG。

```
Canvas 可用? ──Yes──▶ 运行动画
      │
      No
      ▼
  显示静态 SVG
```

### R2: 动画可能影响初始化性能

**风险**: 复杂动画可能占用 CPU，影响初始化速度。

**缓解**:
- 动画在独立的 requestAnimationFrame 循环中运行，不阻塞主线程
- 初始化完成后立即清理动画

### R3: SVG 图标在小尺寸下识别度

**风险**: 简化后的 SVG 在 16x16 或更小尺寸下可能难以识别。

**缓解**:
- 为极小尺寸设计专门的 favicon 版本
- 测试不同尺寸下的显示效果

### T1: 固定配色 vs 主题适配

**权衡**: 选择固定配色而非主题适配。

**理由**:
- 简化实现复杂度
- 黑白配色在深色和浅色背景下都清晰可见
- 品牌一致性优先于主题一致性
