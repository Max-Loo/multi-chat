## Context

当前项目根目录仅有一个中文 `README.md`（~400 行），作为 GitHub 仓库首页展示。项目已支持国际化（i18next，中/英/法三语），但 README 未提供英文版本，不利于国际开发者参与。

## Goals / Non-Goals

**Goals:**

- 将 README 改为双语支持，英文为默认展示版本
- 两个版本相互引用，方便切换
- 在 AGENTS.md 中建立硬约束规则，确保后续维护时两个版本保持同步

**Non-Goals:**

- 不增加 CI 自动检查机制（后续按需添加）
- 不对现有中文内容做修改（仅加顶部链接）
- 不涉及代码层面的国际化改动

## Decisions

### D1: 英文版作为 README.md（默认展示）

**选择**：英文版为 `README.md`，中文版为 `README.zh-CN.md`

**理由**：这是国际开源项目的标准惯例（React、Vue、Next.js 等均采用此方式）。GitHub 自动渲染 `README.md` 为仓库首页，英文默认展示有利于国际社区访问。

**备选方案**：
- 中文默认 + `README_EN.md`：不符合国际惯例，不利于吸引国际贡献者
- `docs/en/README.md`：路径过深，不够直观

### D2: 文件迁移方式

**选择**：将现有 `README.md` 重命名为 `README.zh-CN.md`，然后新建英文版 `README.md`

**理由**：git 会自动追踪重命名，保留完整历史。中文内容原样保留，仅顶部增加语言切换链接。

### D3: AGENTS.md 同步规则为硬约束

**选择**：在文档同步要求中以「必须」措辞写入同步规则

**理由**：双文件一致性需要强制保障，软提醒容易被忽略。规则明确：修改任一版本时，必须同步翻译另一版本，且章节结构必须一致。

## Risks / Trade-offs

- **[内容不同步]** → AGENTS.md 硬约束 + agent 行为保障，后续可视情况增加 CI 检查
- **[AGENTS.md 行数增长]** → 当前 202 行已超 150 行目标，新增约 5 行规则，暂不拆分，后续统一处理
