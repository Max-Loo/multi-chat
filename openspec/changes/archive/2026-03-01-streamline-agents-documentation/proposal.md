## Why

当前 AGENTS.md 文件（1316 行）包含大量详细代码示例、重复说明和可从代码中直接查询的信息，导致：

1. **维护成本高**：代码变更时需要同步更新大量冗余文档
2. **可读性差**：关键信息淹没在大量细节中
3. **内容过时**：详细的测试统计数据（如"2026-03-01 更新"）容易过时
4. **重复内容**：HTTP 插件兼容部分有 3 段几乎相同的说明（750-850 行）

精简 AGENTS.md 可以：
- 保留必要的架构指导和开发规范
- 将详细实现交给代码注释和类型定义
- 通过文档引用提供关键文件位置
- 提升文档的可维护性和可读性

## What Changes

- **删除冗余内容**：移除详细的代码示例、重复说明、过时的测试统计
- **保留核心信息**：项目架构、开发规范、关键约定
- **添加文档引用**：提供关键文件的路径索引，指向具体实现
- **优化结构**：按信息重要性重新组织，提升可读性

## Capabilities

（无新增功能）

### Modified Capabilities
- `agents-guidance`：优化 AGENTS.md 文档结构，提升可维护性和可读性

## Impact

**文档变化**：
- AGENTS.md：从 1316 行精简至约 300-400 行（减少约 70%）

**保留内容**：
1. 项目核心架构（技术栈、目录结构）
2. 开发命令（常用命令）
3. 关键设计说明（InitializationManager、远程模型数据获取、聊天服务层、跨平台兼容性的架构概览）
4. 开发规范（导入路径、代码文档、KISS/YAGNI/DRY/SOLID 原则）
5. 项目约定（时间戳工具、shadcn/ui 安装、文档同步）

**删除内容**：
1. 详细的代码示例（保留简要说明，指向源文件）
2. 重复的 HTTP 插件说明（3 段合并为 1 段）
3. 详细的测试覆盖率统计（保留测试框架信息）
4. 可从 package.json 查询的 Tauri 插件列表
5. 可从代码中查询的 API 使用细节

**新增内容**：
1. **文档维护原则**（开头章节，约 45 行）
    - 内容分类规则（什么该留、什么该删）
    - 添加新内容检查清单（5 个必答问题）
    - 优先级原则（代码注释 → README → AGENTS.md）
    - 维护规范（保持精简、及时更新、定期审查）
    - 正面描述规范（使用"要干xxx"而非"不要干xxx"）
    - 当前文档状态（行数、待优化项、参考方案）
2. **文档参考章节**：提供关键文件的路径索引
