## Context

KeyManagementSetting 页面包含两个功能卡片（密钥导出、重置数据），当前按钮与描述文字在同一行对齐（items-center 仅针对描述行），视觉上按钮位置偏低。应将按钮竖直居中于整个卡片容器内，使其在视觉上更加平衡。

## Goals / Non-Goals

**Goals:**
- 将按钮在整个卡片容器内竖直居中（而非仅与描述文字水平对齐）
- 卡片整体采用左右布局：左侧标题+描述，右侧按钮居中
- 保持现有功能行为不变

**Non-Goals:**
- 不修改对话框（AlertDialog）的布局
- 不改变按钮功能或交互逻辑
- 不引入响应式断点回退（始终使用水平布局）

## Decisions

### 1. 卡片内部布局方案

**选择**: 卡片整体使用 `flex-row items-center justify-between`，左侧区域包含标题和描述（flex-col），右侧按钮自动竖直居中于整个卡片高度

```
┌─────────────────────────────────────┐
│  标题                       [按钮]  │
│  描述文字...                        │
└─────────────────────────────────────┘
```

**替代方案 A**: 描述和按钮在同一行（当前实现）— 按钮仅与描述对齐，不在卡片整体内居中，视觉不平衡。

**替代方案 B**: 完全水平布局（标题和按钮也在同一行）— 标题文字较短时会导致间距过大，且标题与描述的视觉层级会丢失。

### 2. 按钮宽度

**选择**: 移除 `w-full sm:w-auto`，统一使用 `shrink-0` 自适应宽度

**理由**: 水平布局下按钮无需撑满宽度，自适应即可。

## Risks / Trade-offs

- **长描述文本 + 小屏幕**: 描述文字过长时可能压缩按钮空间 → 左侧使用 `flex-1 min-w-0` 确保文字区域可截断，按钮始终可见
