## 概述

本次重构专注于提升测试代码质量，通过引入 MSW（Mock Service Worker）、消除类型不安全、标准化 Fixtures 使用、采用行为驱动测试（BDD）模式，显著降低了测试脆性，提高了测试覆盖率和可维护性。

**重构范围**: 仅修改测试代码，不影响生产代码或应用运行时行为。

## 主要改进

### 1. MSW 全面集成

**改进前**:
- 使用 vi.mock() 隔离外部依赖
- Mock 配置分散在各个测试文件中
- 无法模拟真实 HTTP 请求

**改进后**:
- 创建 16 个标准化的 MSW handlers
- 统一的 MSW 服务器配置
- 支持完整的 HTTP 请求/响应模拟
- 修复 CORS preflight 请求问题

**文件变更**:
- 新增: `src/__test__/msw/handlers/`（16 个 handler 文件）
- 新增: `src/__test__/msw/setup.ts`（MSW 服务器配置）
- 新增: `src/__test__/msw/index.ts`（统一导出）

**成果**:
- 移除 22 处 MSW 迁移相关的 TODO 注释
- 建立 API Mock 标准化流程

### 2. 测试类型安全改进

**改进前**:
- 广泛使用 `any` 类型（336 处未注释使用）
- Mock 对象缺乏类型定义
- 类型错误难以在编译时发现

**改进后**:
- 消除所有未注释的 `any` 类型使用
- 为所有 Mock 对象定义完整的 TypeScript 类型
- 使用 `Mocked<T>` 和 `Partial<T>` 确保类型安全
- 建立类型安全的 Mock 工具函数

**文件变更**:
- 修改: 所有测试文件的类型定义
- 新增: `src/__test__/types/test-types.ts`（测试专用类型）

**成果**:
- `any` 类型使用: 336 → 0（未注释）

- 提升测试代码的类型安全性
