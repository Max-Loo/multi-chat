## Context

`src/__test__/README.md` 是测试辅助工具的核心文档，当前 2085 行。主要问题是同一测试原则在四个章节中重复展开（行为驱动原则、Before/After 示例、反模式列表、最佳实践），以及 AI SDK Mock 的函数级 API 文档占据了 385 行但更适合作为代码内文档。

## Goals / Non-Goals

**Goals:**
- 将 README 从 2085 行精简到约 700 行（减少 66%）
- 消除四个重复章节，合并为一个精简的「测试原则与反模式」章节
- 将 AI SDK Mock 的函数签名和参数文档迁移到 `helpers/mocks/aiSdk.ts` 代码注释
- 删除不适用内容（历史迁移说明、后端通用建议）
- 更新目录结构树以匹配实际文件布局

**Non-Goals:**
- 不重构任何测试代码
- 不修改 helpers 的接口或行为
- 不改动集成测试 README（`src/__test__/integration/README.md`）
- 不重新组织测试目录结构

## Decisions

### 1. 合并策略：四合一 → 精简章节

将「行为驱动测试原则」「Before/After 对比示例」「常见反模式和解决方案」「测试最佳实践」合并为一个「测试原则」章节。

保留内容：核心理念（测试行为而非实现）+ 每种测试类型（组件/Hooks/Redux）各一个正例 + 各一个反例。

删除内容：重复阐述同一原则的 Before/After 完整示例、反模式列表中与正例重复的错误示例、「测试最佳实践」中与前面完全重复的段落。

**理由**：四处内容讲的是同一件事，精简后保留一组正反例足以传达信息。

### 2. AI SDK Mock 文档：README 保留快速示例，细节迁入代码注释

README 仅保留：导入路径、依赖注入用法示例（10-15 行）、注意事项摘要。

`aiSdk.ts` 中为每个导出函数添加 JSDoc：参数说明、返回值、使用示例。

**理由**：385 行的函数级文档在 README 中维护成本高且查阅不便，IDE 悬停代码注释更实用。

### 3. 常见问题排查：精简为问题-方向摘要

将 10 个问题的完整代码示例替换为一行问题描述 + 解决方向。保留问题编号以便引用。

### 4. 删除而非标记

已完成的历史迁移说明直接删除，不保留「已迁移至」标记。后端通用建议同理。

**理由**：git 历史可追溯，文档中保留过时内容的引用反而是噪音。

## Risks / Trade-offs

- [精简后部分细节丢失] → 关键细节已迁移到代码注释或 spec 文件中，不会丢失
- [新人可能缺少详细示例] → 现有测试文件本身就是最好的示例，且核心正反例会保留
