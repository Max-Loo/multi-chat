## Why

随着项目的发展，国际化配置文件中积累了一些未被使用的翻译 key。这些未使用的文案增加了维护负担，可能导致开发者误以为某些功能仍在使用中。通过检查并移除未使用的国际化文案，可以提升代码可维护性，减少混淆，并确保国际化文件与实际代码使用保持同步。

## What Changes

- 扫描所有 TypeScript/TSX 文件中使用的国际化 key（`t()` 函数调用）
- 对比国际化配置文件中定义的所有 key
- 识别并移除所有未使用的 key
- 同步移除英文和中文语言文件中的对应条目
- 验证移除后应用的正常运行

## Capabilities

### New Capabilities
- `i18n-unused-key-removal`: 国际化未使用 key 的检测与移除能力，包括代码扫描、key 使用分析、自动识别和删除未使用条目

### Modified Capabilities
- 无现有能力的需求变更（此为纯粹的代码清理任务）

## Impact

**受影响的文件**:
- `src/locales/en/*.json` - 所有英文语言文件
- `src/locales/zh/*.json` - 所有中文语言文件
- 可能涉及 `src/types/i18n.d.ts` - 类型定义文件（如果需要更新）

**受影响的系统**:
- 国际化系统 (i18n)
- 应用所有使用翻译的页面和组件（移除未使用的 key 不应影响运行时行为）

**风险**:
- 需要确保动态构建的 key（如字符串拼接）不会被误判为未使用
- 需要确保类型系统正确反映移除后的 key 结构
