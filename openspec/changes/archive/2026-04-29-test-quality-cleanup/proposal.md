# 清理伪测试与重写弱断言

## 问题

项目有 177 个测试文件，整体行覆盖率 87.81%，但测试质量存在明显水分：

1. **伪测试**——执行代码但没有有效断言，组件删掉一半功能测试照样通过
2. **重复测试**——同一逻辑用不同 TypeScript 泛型签名再测一遍，运行时走的完全是同一条 JS 路径
3. **弱断言**——断言条件过于宽松，功能完全失效也能通过（如 `toBeLessThanOrEqual`）
4. **守卫空转**——`if` 守卫导致断言从不执行，测试"通过"但什么都没验证

这些伪测试虚增了覆盖率数字，却不提供任何回归保护。在补强测试之前必须先清理干净，否则无法建立可靠的基线。

## 范围

### 要清理的文件

| 文件 | 问题类型 | 具体问题 |
|------|---------|---------|
| `src/__test__/hooks/useDebounce.test.ts` | 重复测试 | 4 个泛型类型测试（string/number/object/array）走完全相同的运行时路径 |
| `src/__test__/components/Layout.test.tsx` | 伪测试 + 重复 | 4 个测试只有 `getByTestId` 无 `expect`；2 个子组件位置测试与已有测试重复 |
| `src/__test__/components/Splitter.test.tsx` | 伪测试 | "应该应用正确的容器样式" 实际未检查任何样式属性 |
| `src/__test__/components/ChatPanel.test.tsx` | 守卫空转 + 重复 | `if (initialValue < 3)` 守卫导致断言可能不执行；3 个结构测试与首测重复 |
| `src/__test__/hooks/useBasicModelTable.test.tsx` | 弱断言 + 重复 | `toBeLessThanOrEqual` 在筛选失效时也能通过；3 模型渲染测试与 2 模型测试走相同路径 |

### 重写原则

- 删除零行为价值的测试（泛型重复、DOM 重复查询）
- 为"渲染不崩就算过"的测试补充具体行为断言
- 弱断言替换为精确断言
- 去掉 `if` 守卫，确保断言必定执行

## 预期结果

- 删除约 10-12 个伪/重复测试用例
- 重写约 5-8 个弱断言测试
- 覆盖率数字可能小幅下降（1-2%），但每个留存的测试都有真实保护力
- 建立干净的基线，为后续分支覆盖补强（变更 2）做好准备

## 非目标

- 不新增测试用例（那是变更 2 的工作）
- 不调整覆盖率阈值
- 不引入新的测试工具或框架
- 不修改被测源代码
